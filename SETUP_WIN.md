# Win 4GBノートを録画専用機にするセットアップ

## アーキテクチャ

```
[Win 4GBノート]                    [MacBook Pro / 任意マシン]
  ├ USB地デジチューナー              ├ transcribe.py
  ├ EDCB（録画予約・録画）           ├ data.json 更新
  └ 録画フォルダを共有               └ アプリでブラウザ閲覧
        ↓ Wi-Fi/SMB                       ↑
        └─────── tsファイル ──────────────┘
```

Win機は **録画だけ**、処理は触りたい時に他マシンで実行。
Win機はメモリ厳しいので「録画ソフト1個だけ動かす」運用。

## 物理セットアップ（写真の壁端子の場合）

```
[壁の「TV」端子]
       │ アンテナケーブル 2m
       ↓
[USB地デジチューナー]
       │ USB
       ↓
[Win 4GBノート]   ←  電源繋ぎっぱなし、24時間 or 録画時間帯起動
```

- ノートPCはこの壁の前に据え置き、移動させない
- スリープしないよう電源設定で「常時オン」に変更
- 自動ログオン設定推奨（再起動でも復帰）

## ソフトインストール（Win側）

Win定番OSS録画スタック「EDCB + TvTest」を使う（軽量・実績豊富）。

### 手順
1. [PX-S1UD用ドライバ](https://www.plex-net.co.jp/) をダウンロード・インストール
2. [TVTest](https://github.com/DBCTRADO/TVTest) をダウンロード・解凍
3. [EDCB](https://github.com/DBCTRADO/EDCB) をダウンロード・解凍
4. BonDriver（PX-S1UD用）配置
5. EpgTimer.exe で予約設定：
   - 毎週月〜土 7:58〜8:32（Eテレ）
   - 番組名検索：「おかあさんといっしょ」「いないいないばあっ!」
6. 出力先フォルダを共有可能な場所に設定（例: `C:\rokuga\`）

> 詳しい設定は「EDCB 録画」「TvTest 設定」でググると無限に解説あり。

### 軽量化Tips
- 録画中はブラウザ・他のアプリ全部閉じる（メモリ確保）
- Windows Update は手動運用に変更（自動再起動防止）
- スリープ・休止状態 OFF
- 不要な常駐アプリ無効化

## ファイル共有（Mac/別マシンから取りに行く）

### 方法A: Windows共有フォルダ（一番シンプル）
1. `C:\rokuga\` を右クリック → プロパティ → 共有 → ネットワーク共有
2. Macから Finder → 移動 → サーバへ接続 → `smb://win-pc-name/rokuga`
3. Macで処理スクリプト実行：
   ```bash
   for f in /Volumes/rokuga/*.ts; do
     python3 transcribe.py "$f" --show okaasan --date $(date +%Y-%m-%d)
   done
   ```

### 方法B: Dropbox / Google Drive 自動アップロード
- 録画フォルダをクラウド同期対象に設定
- どこからでもダウンロードして処理

### 方法C: 自動化（cron + rsync）
- Mac側で毎朝9:00 cron → rsyncでtsファイルをMacに同期 → transcribe.py 実行
- 完了後にWin側のtsを削除（容量節約）

## トラブルシュート

| 症状 | 原因 | 対処 |
|------|------|------|
| TVTestで映らない | ドライバ未認識 / B-CAS未挿入 | デバイスマネージャ確認、B-CASカード再挿入 |
| 受信感度低い（ブロックノイズ） | アンテナ信号弱い | 卓上ブースター追加、または既存レコーダーに分配器 |
| EDCB予約が動かない | EpgDataCap_Bonの起動忘れ | 自動起動設定を確認 |
| 録画ファイル肥大 | 削除運用してない | 処理済みtsを毎朝削除するスクリプト追加 |
| メモリ不足で固まる | 他のアプリ起動中 | Win機は録画専用と割り切る |
